How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?
Building in user needs is done via corresponding with the user and figuring out what they expect the product to do, in this same vain, by extending these user needs we then have user stories.  User stories are built from the users needs and assists in implementing them into the program.  By building out these user stories we also figure out what the expectations are and how to test those expectations for use.
How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?
I use the agile process of six steps, these six steps of agile; By fully using the steps listed here, we can then adhere to the agile process. These are are integrated into tools, such as jira and confluence.
Planning the Project - Setup the project, make sure its management is doable, understand the end goals for a project and figure out the team members needed.
Creating the Product Roadmap - Create a breakdown of the final project, create the story planning points and build out the needed concepts around the primary project concept.
Planning the Release - We then setup the release planning stages, checking to make sure each step is doable in the needed time line.
Planning the Sprint(s) - These are setup for 15 minute sprints, team members can reassess anything as needed. 
Daily Stand-up Meetings -  Daily standup meetings and participants can tell what they accomplished the day before.
Sprint review and retrospective meetings - We then handle the sprint review, making sure the client is shown the final product. We then review the feedback here from the client and users.  On reassessment, these feedback items are then incorporated into the next sprint cycle.
